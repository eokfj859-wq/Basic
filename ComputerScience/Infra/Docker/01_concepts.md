# Docker 핵심 개념 요약
"기존 배포 방식이 서버라는 도화지에 직접 그림을 그리는 것이었다면, 도커는 완성된 그림(이미지)을 액자에 담아 벽에 걸기만 하면 되는 방식이다. 호스트의 커널을 공유하기 때문에 훨씬 가볍고 빠르다."

### 1. 도커를 사용하는 이유
- **환경 일관성**: "내 컴퓨터에선 되는데 서버에선 안 돼"라는 상황 방지.
- **경량화**: VM(가상머신)처럼 OS 전체를 띄우지 않고, 프로세스만 격리해서 실행하므로 훨씬 빠르고 가벼움.

### 2. 3단계 프로세스
1. **Dockerfile**: 이미지를 만들기 위한 '설계도(레시피)'.
2. **Image**: 실행에 필요한 모든 파일이 압축된 '상태(패키지)'. (변경 불가능)
3. **Container**: 이미지를 실행한 '실체(인스턴스)'. (여기서 파일 수정, 네트워크 통신 발생)

### 3. 주요 키워드
- **Port Forwarding**: 호스트(내 컴퓨터) 포트와 컨테이너 포트를 연결 (ex. 8080:80).
- **Volume**: 컨테이너가 삭제되어도 데이터를 유지하기 위해 호스트 저장소와 연결.
